# 本项目 Playwright 自动化测试专属规则
1. 项目结构：
    - 页面模型 POM 放在 pages/ 目录
    - 测试用例放在 tests/ 目录
    - 工具方法放在 utils/ 目录
2. 用例规范：
    - 每个用例包含成功、失败场景
    - 用 async / await 写法
    - 不使用 TS 类型
3. 代码风格：
    - 变量、函数用小驼峰
    - 代码简洁、不冗余
    - 不使用复杂语法
4. 输出要求：
    - 告诉我文件应该放在哪个目录
    - 代码可直接复制运行
    - 不使用 TS、interface、类型定义
5. 提交规范：
    - 提交信息格式：`<类型>: <描述>`
    - 支持的类型：feat（新功能）、fix（修复bug）、docs（文档）、style（格式）、refactor（重构）、chore（构建过程）、revert（版本回退）、perf（性能优化）、test（测试）
    - 提交信息示例：`feat: 增加系统参数管理功能`、`fix: 修复药品出库时没有供应商的问题`
    - 提交前检查：代码通过测试，提交信息符合规范，改动内容与提交信息一致
    - 工具支持：项目已配置 commitlint 和 husky，会自动检查提交信息是否符合规范
